# 🧠 1️⃣ Biggest Idea

👉 last 70 years:

➡️ computer performance **crazy speedல increase ஆயிற்று**

👉 example:

* 1993 supercomputer 💰 $50M
* இன்று phone 📱 $500

➡️ same performance 😳

---

# ⚙️ 2️⃣ Performance improve ஆனது எப்படி?

👉 2 காரணம்:

### 🔹 1. Technology improvement

* transistor improve
* chip size reduce

👉 (Moore’s Law)

---

### 🔹 2. Architecture improvement

* better design ideas
* smart techniques

👉 (இந்த book focus 🔥)

---

# 🚀 3️⃣ Microprocessor revolution

👉 1970s:

➡️ microprocessor வந்தது

👉 result:

* cheap computers
* mass production
* faster growth

---

# 🔥 4️⃣ RISC Revolution (VERY IMPORTANT)

👉 1980s:

➡️ **RISC (Reduced Instruction Set Computer)** வந்தது

👉 effect:

* simple instructions
* faster execution
* better pipeline

---

## 💥 RISC focus:

* pipeline
* cache
* parallelism

👉 இதனால்:

➡️ performance growth = **52% per year** 🔥

---

# ⚔️ 5️⃣ Old vs New architectures

* RISC systems ✅

👉 Intel என்ன பண்ணுச்சு?

➡️ x86 → internally RISC போல convert பண்ணுது

# 📉 8️⃣ Problem starts (IMPORTANT 🔥)

👉 2003 பிறகு:

➡️ performance slow ஆகுது

---

## ⚠️ காரணங்கள்:

### ❌ Dennard Scaling ended

👉 power reduce ஆக முடியல

---

### ❌ Moore’s Law slow ஆகுது

👉 transistor growth slow

---

# 🔄 9️⃣ Solution: Multicore

👉 single CPU fast பண்ண முடியல

➡️ multiple cores use பண்ண ஆரம்பிச்சாங்க

---

## Parallelism types:

* ILP → instruction level
* DLP → data level
* TLP → thread level
* RLP → request level

---

# ⚠️ 10️⃣ Amdahl’s Law

👉 important limitation:

➡️ programல 10% serial இருந்தா:

👉 max speedup = 10x மட்டும்
